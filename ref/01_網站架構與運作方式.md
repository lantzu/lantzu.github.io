# 網站架構與運作方式 (Ueno 風格)

本文件詳細描述 `lantzu.github.io` 的網站架構、設計理念以及自動化工作流。

## 1. 設計理念
參考 WordPress [Ueno](https://wordpress.com/zh-tw/theme/ueno) 主題，本網站採用 **極簡主義 (Minimalism)** 與 **結構化 (Structured)** 的設計風格：
- **視覺風格**：大膽的排版（Typography）、充足的留白（Whitespace）、中性的色調（如深灰、淺灰、黑、白）。
- **字體選用**：偏好等寬字體（Monospace）或現代無襯線字體（Sans-serif），強調現代感與技術感。
- **佈局**：響應式設計，首頁採用非對稱或兩欄式的網格佈局來展示文章。

## 2. 網站架構 (Sitemap)
- **首頁 (Home)**：展示最新的精選文章或專案，包含大標題（Hero Section）與文章列表網格（Query Loop Grid）。
- **文章分類頁 (Category/Archive)**：依據文章屬性（如：技術、生活、攝影）進行分類展示。
- **文章內頁 (Single Post)**：包含標題、日期、分類、主文內容、以及嵌入的照片。
- **關於 (About)**：簡單的個人介紹與聯繫方式。

## 3. 目錄結構
```text
lantzu.github.io/
├── index.html          # 網站首頁
├── css/                # 樣式表 (Ueno 風格定義)
├── js/                 # 網站邏輯 (如分類切換)
├── assets/             # 網站靜態資源 (Icon, Logo 等)
├── posts/              # 生成後的文章 HTML 頁面
├── photos/             # 生成後供網頁讀取的照片 (最佳化後)
└── ref/                # 原始資料夾 (工作流核心)
    ├── 00_主架構.txt    # 原始需求說明
    ├── 01_網站架構...md # 本文件
    ├── categories/     # 文章分類目錄 (如: tech, life)
    │   └── [category]/
    │       ├── article_v1.txt  # 原始文字檔
    │       └── image1.jpg      # 原始照片檔
    └── global_config.json      # 網站全域設定 (標題、社群連結等)
```

## 4. 自動化工作流 (Workflow)
為了簡化維護，我們採用「資料夾驅動」的生成方式：

### 步驟 A：撰寫內容 (User 操作)
1. 在 `ref/categories/[分類名稱]/` 下建立一個 `.txt` 檔案。
2. 在文字檔中撰寫內容，並使用特定語法標註照片位置（例如：`[[photo: image1.jpg]]`）。
3. 將對應的照片放入同一個目錄中。

### 步驟 B：生成網頁 (Gemini 操作)
1. **掃描**：我會讀取 `ref/` 下的所有分類與文字檔。
2. **解析**：解析 `.txt` 中的標題、日期、內容與照片標記。
3. **處理照片**：將照片移動/複製到 `photos/` 目錄，並確保路徑正確。
4. **渲染**：根據 Ueno 風格的 HTML 模板，生成 `index.html` (更新列表) 與 `posts/[文章名稱].html`。
5. **更新索引**：自動將新文章加入首頁的網格列表中，並依分類歸檔。

### 步驟 C：發佈 (User 操作)
- 確認生成結果無誤後，由您執行 `git add`, `commit` 與 `push` 到 GitHub。
